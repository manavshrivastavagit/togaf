# Landscape Tier: Capability Architectures

Capability Architectures provide detailed blueprints for cross-cutting horizontal capabilities that are shared across multiple product segments (e.g., enterprise payment hubs, identity provider engines).

## Contents
This folder acts as a placeholder for capability blueprints, such as:
- **Enterprise Identity & Access Management (IAM) Blueprint**: Governing Keycloak/OIDC configs shared across Retail, Corporate, and Lending.
- **Enterprise Messaging Standard (Kafka)**: Governing message schemas, partition policies, and producer/consumer requirements.
- **Host-to-Host Sponsor Banking Gateways**: Reusable payment connectivity templates.

## Usage Guidelines
Capability architectures are maintained by horizontal platform engineering teams and must be conformed to by all stream-aligned product engineering teams.
